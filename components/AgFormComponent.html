Description

Componente principal para manejar formularios en la aplicación.

Este componente incluye múltiples subcomponentes para la selección, entrada, modales, búsqueda y tablas, así como servicios y utilidades necesarias para la funcionalidad del formulario.

Convierte una URL de datos (data URI) en un objeto Blob.

Este método toma una URL de datos que representa un archivo en formato base64 y lo convierte en un Blob, que puede ser utilizado para manipular el archivo en aplicaciones web, como subirlo a un servidor.

El proceso incluye:

  • Decodificar la cadena base64 en una cadena de bytes.
  • Extraer el tipo MIME de la URL de datos.
  • Crear un ArrayBuffer y un Uint8Array para almacenar los bytes del archivo.
  • Convertir los bytes en un objeto Blob con el tipo MIME adecuado.

Inicializa el formulario y configura las estructuras relacionadas.

Este método realiza varias tareas para configurar el formulario principal y las relaciones relacionadas. Incluye la inicialización de variables, la configuración de form_object y la obtención de datos relacionados desde la API.

  • Inicializa form_object_relational y select_target_list como arreglos vacíos.
  • Itera sobre apiResponse.content para configurar el objeto del formulario (form_object) y marca cada ítem como válido.

  • Solicita datos relacionados a la API para obtener definiciones de formularios relacionados.
  • Itera sobre la respuesta de la API (data_relation) para configurar esquemas de formularios relacionados y almacenar las relaciones.
  • Usa el método setItemSchemaFormObject para definir la estructura de cada esquema de formulario relacionado.
  • Actualiza form_object_relational_schemas con las definiciones y esquemas de formularios relacionados.

  • Marca el formulario como listo estableciendo form_ready en true.

Configura el objeto del formulario según el esquema del ítem.

Este método ajusta el objeto de parámetros del formulario (obj_param) en función de las características del ítem proporcionado. También actualiza las listas de elementos editables y objetivos de selección según las opciones del ítem.

  • Si el tipo del ítem es tinyint, añade el nombre del ítem a boolean_fields.
  • Si el tipo del ítem es numérico, inicializa el valor en obj_param con 0. De lo contrario, lo inicializa con una cadena vacía.
  • Si las opciones del ítem son 'editable', añade el ítem a la lista editable_list.
  • Si las opciones del ítem no son nulas, no son 'blank', y no son 'editable', se parsea el valor de item.options y se utiliza para actualizar api_res_content y obj_param.
  • Si el valor de mini_opt tiene una propiedad value definida, se añade al objeto obj_param con una cadena vacía.
  • Si mini_opt.type es 'select', se asegura que el objeto obj_param tenga una cadena vacía para el nombre del ítem.
  • Si mini_opt.target está definido, se añade un objeto con el nombre del ítem y el índice de destino a select_target_list.

Establece un valor de selección en el objeto del formulario y ejecuta una función de callback si está disponible.

Este método asigna el valor del evento recibido al campo del formulario especificado por index_name. Luego, verifica si se proporciona una función de callback en options.
