Valida el formulario y procesa los datos antes de enviarlos a la API.

Este método realiza varias tareas para validar y procesar el formulario antes de enviarlo:

  • Convierte los valores booleanos en el objeto del formulario a 1 o 0.

  • Valida los campos obligatorios, mostrando un mensaje de error si algún campo requerido está vacío.

      "schema": form_object_schema
    });

    data_relation[index].relationals = [];

 * 
 * Este método ajusta el objeto de parámetros del formulario (`obj_param`) en función de las
 * características del ítem proporcionado. También actualiza las listas de elementos editables y
 * objetivos de selección según las opciones del ítem.
 * 
 * - Si el tipo del ítem es `tinyint`, añade el nombre del ítem a `boolean_fields`.
 * - Si el tipo del ítem es numérico, inicializa el valor en `obj_param` con `0`. De lo contrario, lo inicializa con una cadena vacía.
 * - Si las opciones del ítem son `'editable'`, añade el ítem a la lista `editable_list`.
 * - Si las opciones del ítem no son nulas, no son `'blank'`, y no son `'editable'`, se parsea el valor de `item.options` y se utiliza para actualizar `api_res_content` y `obj_param`.
 * - Si el valor de `mini_opt` tiene una propiedad `value` definida, se añade al objeto `obj_param` con una cadena vacía.
 * - Si `mini_opt.type` es `'select'`, se asegura que el objeto `obj_param` tenga una cadena vacía para el nombre del ítem.
 * - Si `mini_opt.target` está definido, se añade un objeto con el nombre del ítem y el índice de destino a `select_target_list`.
 * 
 * @param obj_param - Objeto que se está configurando con los datos del ítem.
 * @param api_res_content - Contenido de la respuesta de la API que se está actualizando con las opciones parseadas.
 * @param item - El ítem cuyo esquema se está aplicando.
 * @param contador - Índice que se utiliza para acceder al contenido de `api_res_content`.
 * 
 * @memberof AgFormComponent
 */
setItemSchemaFormObject(objParam: any, apiResponseContent: any, item: any, index: number): void {
  // Manejo de campos de tipo booleano
  if (item.type === "tinyint") {
    this.boolean_fields.push(item.name);
  }
  // Inicialización del campo según su tipo
  objParam[item.name] = this.isNumberType(item.type) ? 0 : "";
  // Manejo de campos editables
  if (item.options === "editable") {
    this.editable_list.push({ name: item.name, type: item.type });
    return; // Salir de la función ya que no es necesario procesar más
  }
  // Manejo de otras opciones
  if (item.options && item.options !== "blank") {
    const parsedOptions = JSON.parse(item.options);
    apiResponseContent[index]["parsedOptions"] = parsedOptions;

    if (parsedOptions.value) {
      objParam[`${item.name}${parsedOptions.value}`] = "";
    }
    if (parsedOptions.type === "select") {
      objParam[item.name] = "";
    }
    if (parsedOptions.target !== undefined) {
      this.select_target_list.push({
        option_container: item.name,
        target: parsedOptions.index_target,
      });
    }

  }
}

 * predefinidos: "integer", "double", "int", "bigint" o "decimal".
 * 
 * @param item_type - El tipo de dato a verificar.
 * @returns `true` si el tipo de dato es uno de los tipos numéricos definidos; `false` en caso contrario.
 * 
 * @memberof AgFormComponent
 */
isNumberType(item_type: string) {
  return item_type === "integer" || item_type === "double" || item_type === "int" || item_type === "bigint" || item_type === "decimal";
}


  /**
 * Convierte todos los valores booleanos en un objeto a números.
 * 
 * Este método itera sobre todas las propiedades del objeto proporcionado. Para cada propiedad que tenga un valor booleano, el valor se convierte en 1 si es `true` o en 0 si es `false`. 
 * Esta conversión asegura que los valores booleanos sean representados como números en lugar de como valores booleanos, facilitando su procesamiento en contextos que requieren números.
 * 
 * @param {any} obj - El objeto cuyo valor booleano se convertirá a números.
 * 
 * @returns {any} - El objeto modificado con valores booleanos convertidos a números.
 * @memberof AgFormComponent
 */
convertBooleansToNumbers(obj: any) {
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      if (typeof obj[key] === "boolean") {
        obj[key] = obj[key] ? 1 : 0;
      }
    }
  }
  return obj;
}


  /**
 * Valida el formulario y procesa los datos antes de enviarlos a la API.
 * 
 * Este método realiza varias tareas para validar y procesar el formulario antes de enviarlo:
 * - Convierte los valores booleanos en el objeto del formulario a 1 o 0.
 * - Valida los campos obligatorios, mostrando un mensaje de error si algún campo requerido está vacío.
 * - Procesa los datos relacionales, manejando validaciones anidadas para relaciones múltiples.
 * - Prepara los datos relacionales para ser enviados en la solicitud a la API.
 * - Configura la solicitud API basada en la acción especificada (crear, actualizar o eliminar).
 * - Envía la solicitud API utilizando el método HTTP apropiado (POST, PUT, DELETE).
 * 
 * @param {string} [action] - La acción a realizar, que puede ser 'update' para actualizar, 'destroy' para eliminar, 
 *                            o undefined para crear un nuevo registro.
 * @returns {void}
 * @memberof AgFormComponent
 */
validateForm(action?: String) {
  this.convertBooleansToNumbersInFormObject();

  const isMainFormValid = this.formValidationService.validateMainForm(this.apiResponse, this.form_object);
  const areRelationalFormsValid = this.formValidationService.validateRelationalForms(this.relationalData, this.form_object_relational);

  if (!isMainFormValid || !areRelationalFormsValid) return;

  this.prepareFormObjectForSubmission();
  this.submitForm(action);
}

// Convierte booleanos a números en el form_object
private convertBooleansToNumbersInFormObject() {
  this.boolean_fields.forEach((field) => {
    this.form_object[field] = this.form_object[field] ? 1 : 0;
  });
}

// Prepara el form_object para el envío
private prepareFormObjectForSubmission() {
  this.form_object["relationable"] = [];
  this.form_object_relational.forEach((relationalData, index) => {
    const relationItem: { [key: string]: any } = {
      relation: this.relationalData[index].item_name,
      editable: this.relationalData[index].editable,
      relation_id: this.relationalData[index].relation_id,
      data: relationalData.map(this.convertBooleansToNumbers)
    };
    this.form_object["relationable"].push(relationItem);
  });

  this.editable_list.forEach((element: any) => {
    if (element.type === "date" || element.type === "datetime") {
      this.form_object[element.name] = new Date();
    }
    if (element.type === "tinyint") {
      this.form_object[element.name] = this.form_object[element.name] == 1 ? "SI" : "NO";
    }
  });
}

// Envía el formulario al backend con manejo de errores mejorado
private submitForm(action?: String) {
  let final_endpoint = this.buildEndpoint(action);
  const method = this.determineHttpMethod(action);
  let request;

  switch (method) {
    case HttpMethod.POST:
      request = this.apiService.post(final_endpoint, this.form_object);
      break;
    case HttpMethod.PUT:
      request = this.apiService.put(final_endpoint, this.form_object);
      break;
    case HttpMethod.DELETE:
      request = this.apiService.delete(final_endpoint, this.form_object);
      break;
    default:
      throw new Error("Método HTTP no soportado");
  }

  request.subscribe(
    (data: any) => this.handleSuccessResponse(data),
    (error: any) => this.handleErrorResponse(error)
  );

  this.isPopUpAlert = true;
}

// Construye el endpoint final basado en la acción
private buildEndpoint(action?: String): string {
  if (this.crud) {
    if (action === "update") {
      return `${this.endpoint}/${this.form_object["id"]}/update`;
    } else if (action === "destroy") {
      return `${this.endpoint}/${this.form_object["id"]}/delete`;
    }
    return `${this.endpoint}/create`;
  }
  
  return `${this.endpoint}`;

}

// Determina el método HTTP a utilizar basado en la acción
private determineHttpMethod(action?: String): HttpMethod {
  if (this.crud && this.formHasID()) {
    if (action === "update") {
      return HttpMethod.PUT;
    } else if (action === "destroy") {
      return HttpMethod.DELETE;
    }
  }
  return HttpMethod.POST;
}

// Maneja la respuesta exitosa del servidor
private handleSuccessResponse(data: any) {
  this.dataLoaded.emit(data);
  this.message = data.message || "Exito al guardar";
  this.color = "#105b10";

  if (data.relation_type === "NN" || data.relation_type === "nn") {
    this.createIntermediateTable(data.id);
  }

  this.cleanForm();
}

// Maneja la respuesta de error del servidor
private handleErrorResponse(error: any) {
  this.logError(error);
  this.message = this.getErrorMessage(error);
  this.color = "#801111";
}

  });

  /**
 * Agrega ítems relacionados al formulario basándose en un identificador de relación.
 * 
 * Este método primero verifica si el campo 'id' en `form_object` está vacío o es igual a 0; si es así, no realiza ninguna acción y retorna inmediatamente. 
 * Si el campo 'id' tiene un valor válido, el método realiza una solicitud a la API para obtener los ítems relacionados usando el `relation_id` proporcionado y el `id` del formulario. 
 * Una vez recibida la respuesta, el método itera sobre los datos obtenidos y llama al método `addRelationalItem` para agregar cada ítem relacional a `relationalData` en el índice especificado.
 * 
 * @param {number} relation_id - El identificador de la relación para la cual se deben obtener los ítems relacionados.
 * @param {number} index - El índice en `relationalData` donde se deben agregar los ítems relacionados.
 * 
 * @returns {void}
 * @memberof AgFormComponent
 */
public addRelationalItem(relation_definition_item: any, index: number, relational_form_data?: any) {
  const clonedFields = this.cloneRelationalFields(relation_definition_item.fields);
  this.ensureRelationalsExist(index);
  this.relationalData[index].relationals.push(clonedFields);
  const clonedSchema = this.getClonedSchema(relation_definition_item, relational_form_data);
  this.addClonedSchemaToForm(index, clonedSchema);
}

// Clona los campos relacionales y los valida
private cloneRelationalFields(fields: any[]): any[] {
  return fields.map((field) => {
    field.isValid = true;
    return { ...field };
  });
}

// Asegura que la lista de relationals exista para el índice dado
private ensureRelationalsExist(index: number) {
  if (!this.relationalData[index].relationals) {
    this.relationalData[index].relationals = [];
  }
}

// Obtiene un esquema clonado basado en la definición del formulario relacional
private getClonedSchema(relation_definition_item: any, relational_form_data?: any): any {
  const schemaIndex = this.form_object_relational_schemas.findIndex(
    (schema) => schema["definition"] === relation_definition_item.form_definition
  );

  if (schemaIndex === -1) return null;

  if (relational_form_data) {
    return { ...relational_form_data };
  } else {
    return { ...this.form_object_relational_schemas[schemaIndex]["schema"] };
  }
}

// Agrega el esquema clonado al formulario relacional en el índice dado
private addClonedSchemaToForm(index: number, clonedSchema: any) {
  if (!clonedSchema) return;

  if (!this.form_object_relational[index]) {
    this.form_object_relational[index] = [];
  }
  this.form_object_relational[index].push(clonedSchema);
}

  /**
 * Asigna un valor al objeto de formulario (`form_object`) en función del evento y las opciones proporcionadas.
 * 
 * Este método se encarga de asignar un valor a una propiedad del objeto `form_object` basado en la 
 * interacción del usuario (evento). Si el evento contiene un subevento, se utiliza este subevento para 
 * la asignación. Luego, se procesa el texto de las opciones eliminando signos de `+` y espacios, 
 * y reemplazando las variables en el texto con los valores correspondientes del evento.
 * 
 * - Si el evento contiene un subevento, se utiliza este subevento para la asignación.
 * - El valor correspondiente al nombre (`name`) y a la clave `options.value` se asigna en `form_object`.
 * - El texto en `options.text` es procesado para remover signos `+` y espacios en blanco alrededor de ellos.
 * - Las variables en el texto de la opción son reemplazadas por los valores del evento.
 * - Se actualiza el `form_object` con el texto procesado y se fuerza una detección de cambios en la interfaz de usuario.
 * 
 * @param name — El nombre de la propiedad del objeto `form_object` a la que se asignará el valor.
 * @param event — El evento que contiene los datos para la asignación. Puede incluir un subevento.
 * @param options — Objeto que contiene las opciones de texto y valor para la asignación.
 * 
 * @memberof AgFormComponent
 */
asignarValue(name: string, event: any, options: any) {
  // Validación inicial
  if (!name || !event || !options) {
    console.error("Invalid parameters passed to asignarValue:", { name, event, options });
    return;
  }

  // Extracción del valor
  
  let value = event[options.value];

  if (value === undefined) {
    value = event.event[options.value];
    if (value === undefined) {

      console.warn(`Value for ${options.value} not found in event:`, event);
      return;
    }
  }

  // Asignación del valor
  this.form_object[name] = value;

  // Limpieza del texto y reemplazo de variables en la plantilla
  const cleanedText = this.cleanAndReplaceText(options.text, event.event);

  // Asignación del texto procesado
  this.form_object[`${name}${options.value}`] = cleanedText;
  this.cdr.detectChanges();
}

private cleanAndReplaceText(template: string, context: any): string {
  // Remueve los signos + y espacios alrededor de ellos
  const cleanedTemplate = template.replace(/\s*\+\s*/g, "");

  // Reemplaza las variables dentro del template con los valores del contexto
  const replacedText = cleanedTemplate.replace(/(\w+)/g, (match) => {
    return context[match] !== undefined ? context[match] : match;
  });

  // Remueve cualquier comilla restante
  return replacedText.replace(/[""]/g, "");
}

  /**
 * Maneja el cambio de archivo en un campo de formulario.
 * 
 * Este método se activa cuando un archivo es seleccionado. Realiza las siguientes acciones:
 * 1. Obtiene el archivo desde el evento.
 * 2. Inicializa el campo del formulario para almacenar una cadena vacía en lugar de una URL de datos en base64.
 * 3. Si el archivo está presente, usa un `FileReader` para leer el archivo y convertirlo en una URL de datos.
 * 4. Guarda la vista previa de la imagen si es necesario.
 * 5. Convierte la URL de datos en un Blob y la almacena en el campo del formulario como una URL de datos.
 * 
 * @param {any} event - El evento que contiene el archivo.
 * @param {number} index - El índice para almacenar la vista previa de la imagen.
 * @param {string} fieldName - El nombre del campo en el objeto del formulario donde se almacenará el archivo.
 * 
 * @returns {void}
 * @memberof AgFormComponent
 */
async onFileChange(event: any, index: number, fieldName: string) {
  const file: File = event.event;

  // Validaciones adicionales
  if (!this.isValidFileType(file)) {
    this.snackBar.error("Tipo de archivo no soportado.");
    return;
  }

  if (!this.isValidFileSize(file)) {
    this.snackBar.error("El archivo es demasiado grande.");
    return;
  }

  this.form_object[fieldName + "_64"] = "";

  if (file) {
    try {
      const dataUrl = await this.readFileAsDataURL(file);
      this.imagePreview![index] = dataUrl;
      this.form_object[fieldName + "_64"] = dataUrl;

      this.snackBar.success("Archivo cargado correctamente.");
    } catch (error) {
      this.snackBar.error("Error al procesar el archivo.");
      console.error("Error leyendo el archivo:", error);
    }
  }
}

private isValidFileType(file: File): boolean {
  const allowedTypes = ["image/jpeg", "image/png", "application/pdf"];
  return allowedTypes.includes(file.type);
}

private isValidFileSize(file: File): boolean {
  const maxSizeInBytes = 5 * 1024 * 1024; // 5 MB
  return file.size <= maxSizeInBytes;
}

private readFileAsDataURL(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();

    reader.onload = () => resolve(reader.result as string);
    reader.onerror = reject;

    reader.readAsDataURL(file);
  });
}


  /**
 * Convierte una URL de datos (data URI) en un objeto Blob.
 * 
 * Este método toma una URL de datos que representa un archivo en formato base64 y lo convierte en un Blob, 
 * que puede ser utilizado para manipular el archivo en aplicaciones web, como subirlo a un servidor.
 * 
 * El proceso incluye:
 * - Decodificar la cadena base64 en una cadena de bytes.
 * - Extraer el tipo MIME de la URL de datos.
 * - Crear un `ArrayBuffer` y un `Uint8Array` para almacenar los bytes del archivo.
 * - Convertir los bytes en un objeto Blob con el tipo MIME adecuado.
 * 
 * @param {string} dataURI - La URL de datos que representa el archivo en formato base64.
 * 
 * @returns {Blob} - El objeto Blob que representa el archivo.
 * @memberof AgFormComponent
 */
dataURItoBlob(dataURI: string): Blob | null {
  try {
    // Validación de formato de dataURI
    if (!dataURI.includes(",")) {
      throw new Error("Invalid dataURI format.");
    }

    const byteString = atob(dataURI.split(",")[1]);
    const mimeString = dataURI.split(",")[0].split(":")[1].split(";")[0];

    const ab = new ArrayBuffer(byteString.length);
    const ia = new Uint8Array(ab);
    for (let i = 0; i < byteString.length; i++) {
      ia[i] = byteString.charCodeAt(i);
    }

    return new Blob([ab], { type: mimeString });
  } catch (error) {
    console.error("Error converting dataURI to Blob:", error);
    return null;
  }
}

 * Elimina la imagen de vista previa y limpia el campo correspondiente en el objeto del formulario.

 * Si se proporciona, busca la función correspondiente en el objeto `functions` utilizando el nombre de la función de callback. Si la función existe, la llama con los parámetros necesarios, 
 * que incluyen el índice del objetivo y el valor del evento. Finalmente, se llama a `cdr.detectChanges()` para actualizar la vista. Si no se encuentra la función de callback, se muestra un mensaje de error en la consola.
 * 
 * @param {any} event - El evento que contiene el valor que se debe asignar al campo del formulario.
 * @param {string} index_name - El nombre del campo en el objeto del formulario donde se debe establecer el valor.
 * @param {any} options - Opciones que pueden incluir un callback y otros parámetros para la función de callback.
 * 
 * @returns {void}
 * @memberof AgFormComponent
 */
setSelectValue(event: any, index_name: string, options: any) {
  // Validaciones iniciales
  if (!event || !index_name || !options) {
    console.error("Invalid parameters passed to setSelectValue:", { event, index_name, options });
    return;
  }

  // Asignar el valor al form_object
  this.form_object[index_name] = event.event;

  // Validar si hay un callback definido
  if (!options.callback) {
    console.warn(`No callback defined for ${index_name}.`);
    return;
  }

  // Obtener la función de callback
  const func = this.functions[options.callback];

  // Validar si la función de callback es válida
  if (typeof func !== "function") {
    console.error(`Callback function ${options.callback} not found or not a function.`);
    return;
  }

  try {
    // Ejecutar la función de callback con los parámetros necesarios
    func(options.index_target, event.event);
    this.cdr.detectChanges();
  } catch (error) {
    console.error(`Error executing callback ${options.callback}:`, error);
  }
}
